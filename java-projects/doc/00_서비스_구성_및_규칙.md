## 1. 서비스 개요

### 1.1 서비스 설명

이 서비스는 **게시판 시스템**과 **십자낱말 맞추기 게임**을 통합한 Spring Boot 기반 웹 애플리케이션입니다.

#### 주요 기능
- **게시판 시스템**: 다양한 주제의 게시판에서 글 작성, 조회, 댓글, 추천 등의 기능 제공
- **십자낱말 게임**: 레벨별 난이도로 구성된 십자낱말 퍼즐 게임
- **테마별 게임**: K-POP, K-MOVIE, K-DRAMA, K-Culture 등 테마별 퍼즐 게임
- **관리자 시스템**: 단어 관리, 템플릿 관리, 힌트 생성, 통계 조회 등

#### 기술 스택
- **Backend**: Spring Boot 3.5.5 (Java 17)
- **Database**: PostgreSQL
- **Session**: Redis (Spring Session)
- **Security**: Spring Security (세션 기반 인증)
- **Template Engine**: Thymeleaf
- **Build Tool**: Gradle 8.14.3

#### 서버 정보
- **포트**: 8080
- **프로젝트 경로**: `/var/www/html/java-projects/`
- **데이터베이스**: PostgreSQL (127.0.0.1:5432)

## 3. 각 서비스 구성

### 3.1 게시판 서비스 (Board Service)

#### 기능
- 게시판 목록 조회 (페이지네이션)
- 글 작성, 수정, 삭제
- 댓글 작성 및 관리
- 추천/비추천 기능
- 검색 기능
- 게시판 타입별 분류

#### 주요 클래스
- **BoardController**: 게시판 요청 처리
  - `GET /board`: 게시판 메인 페이지
  - `GET /board/{boardType}`: 특정 게시판 목록
  - `GET /board/{boardType}/create`: 글 작성 페이지
  - `POST /board/{boardType}/create`: 글 작성 처리
  - `GET /board/{boardType}/{id}`: 글 상세 조회
  - `POST /board/{boardType}/{id}/comment`: 댓글 작성

- **BoardRepository**: 게시판 데이터 접근
  - `findByBoardTypeOrderByCreatedAtDesc`: 게시판별 최신순 조회
  - `findByBoardTypeAndSearch`: 검색 기능

- **Board Entity**: 게시판 엔티티
  - `title`: 제목
  - `content`: 내용
  - `user`: 작성자
  - `boardType`: 게시판 타입
  - `views`: 조회수
  - `parent`: 부모 글 (답글용)

#### 인증 요구사항
- 일부 게시판은 인증 필요 (`requiresAuth` 플래그)
- 인증이 필요한 게시판은 로그인 페이지로 리다이렉트

## 4. 퍼즐 게임(단어와 템플릿) 구성 알고리즘

### 4.1 개요

퍼즐 게임은 **템플릿 기반 단어 추출 알고리즘**을 사용하여 동적으로 십자낱말 퍼즐을 생성합니다.

#### 핵심 개념
- **템플릿 (Template)**: 그리드 패턴과 단어 위치 정보를 포함한 퍼즐 구조
- **단어 추출 (Word Extraction)**: 템플릿의 각 위치에 맞는 단어를 데이터베이스에서 선택
- **교차점 처리 (Intersection)**: 여러 단어가 교차하는 지점의 음절 일치 처리

---

### 4.2 템플릿 구조

#### 템플릿 정보
- **id**: 템플릿 고유 ID
- **level_id**: 레벨 ID (난이도)
- **grid_pattern**: 그리드 패턴 (2차원 배열, 0=빈칸, 1=글자칸, 2=검은칸)
- **word_positions**: 단어 위치 정보 (JSON)
  - `id`: 단어 위치 ID
  - `start_x`, `start_y`: 시작 좌표
  - `length`: 단어 길이
  - `direction`: 방향 ("across" 또는 "down")

#### 템플릿 선택
- 레벨별로 활성화된 템플릿 중 랜덤 선택
- 실패 시 다른 템플릿으로 재시도 (최대 5회)

---

### 4.3 단어 추출 알고리즘

#### 전체 흐름

```
1. 템플릿 선택
   ↓
2. 템플릿의 word_positions 순회
   ↓
3. 각 단어 위치에 대해:
   ├─ 교차점이 없는 경우 → 독립 단어 추출
   └─ 교차점이 있는 경우 → 교차점 단어 추출
   ↓
4. 모든 단어 추출 성공 시 완료
   └─ 실패 시 다른 템플릿으로 재시도 (최대 5회)
```

#### 1단계: 독립 단어 추출 (No Intersection)

**조건**: 다른 단어와 교차점이 없는 단어

**알고리즘**:
```java
1. 단어 길이(length) 확인
2. 난이도 조건 확인 (wordDifficulty, hintDifficulty)
3. 테마 조건 확인 (cat2, 선택적)
4. 데이터베이스에서 조건에 맞는 단어 조회:
   - is_active = true
   - length = 단어 길이
   - difficulty = wordDifficulty
   - cat2 = theme (테마별 퍼즐인 경우)
5. 랜덤 선택
6. 힌트 조회 (hintDifficulty에 맞는 힌트)
7. 확정 단어 목록에 추가
```

**SQL 예시**:
```sql
SELECT * FROM pz_words 
WHERE is_active = true 
  AND length = :length
  AND difficulty = :wordDifficulty
  AND cat2 = :theme  -- 테마별 퍼즐인 경우
ORDER BY RANDOM() 
LIMIT 1
```

#### 2단계: 교차점 단어 추출 (Intersection Word)

**조건**: 이미 확정된 단어와 교차점이 있는 단어

**알고리즘**:
```java
1. 교차점 찾기:
   - 현재 단어 위치와 확정된 단어들의 위치 비교
   - 교차하는 좌표 계산
   
2. 교차점 음절 추출:
   - 교차점에서 확정된 단어의 음절 추출
   - 현재 단어에서 해당 음절의 위치 계산
   
3. 조건에 맞는 단어 조회:
   - 단어 길이 = length
   - 특정 위치의 음절 = 교차점 음절
   - 난이도 조건
   - 테마 조건 (선택적)
   
4. 데이터베이스에서 조회:
   - 단어의 특정 위치 음절이 교차점 음절과 일치
   - 다른 조건들도 만족
   
5. 랜덤 선택
6. 힌트 조회
7. 확정 단어 목록에 추가
```

**교차점 계산 예시**:
```
확정된 단어: "가나다라" (가로, 위치: (0,0))
현재 단어: (세로, 위치: (1,0), 길이: 4)

교차점: (1,0)에서 "나" 음절
→ 현재 단어의 2번째 위치에 "나"가 와야 함
→ "?나??" 형태의 단어 검색
```

**SQL 예시**:
```sql
SELECT * FROM pz_words 
WHERE is_active = true 
  AND length = :length
  AND difficulty = :wordDifficulty
  AND cat2 = :theme
  AND SUBSTRING(word, :syllablePosition, 1) = :intersectionSyllable
ORDER BY RANDOM() 
LIMIT 1
```

#### 3단계: 재시도 로직

**실패 시 처리**:
1. 단어 추출 실패 시 실패 로그 저장
2. 다른 템플릿 선택하여 재시도
3. 최대 5회 재시도
4. 모든 재시도 실패 시 에러 반환

**실패 로그 저장**:
- `FailedWordExtraction` 테이블에 저장
- 템플릿 ID, 레벨, 난이도, 단어 위치 정보 등 저장
- 추후 분석 및 개선에 활용

### 4.4 테마별 퍼즐 생성 알고리즘

#### 특수 처리

테마별 퍼즐은 일반 퍼즐과 동일한 알고리즘을 사용하되, **테마 조건(cat2)**을 추가로 적용합니다.

**흐름**:
```
1. 테마 선택 (예: "K-POP")
   ↓
2. 레벨 1 템플릿 랜덤 선택
   ↓
3. 템플릿 기반 단어 추출 (cat2 = 'K-POP' 조건 추가)
   ├─ 독립 단어: cat2 = 'K-POP' 조건
   └─ 교차점 단어: cat2 = 'K-POP' 조건
   ↓
4. 힌트 조회
   ↓
5. 퍼즐 데이터 구성
```

**테마 조건 적용**:
- 모든 단어 추출 시 `cat2 = theme` 조건 추가
- 테마에 맞는 단어만 선택
- 교차점 처리도 테마 조건을 만족하는 단어만 선택
