# 퍼즐게임-게시판 통합작업 세부내역

## 📋 작업 개요

### 요청사항
- **목표**: 8080(게시판)과 8081(퍼즐게임) Spring Boot 애플리케이션을 단일 애플리케이션(8080)으로 통합
- **도메인**: `natus250601.viewdns.net`
- **라우팅 구조**:
  - 메인 화면: `/` (기존 퍼즐게임 메인과 동일)
  - 게시판: `/board`
  - 퍼즐게임: `/Korean/game`, `/K-pop`, `/K-movie`, `/K-Drama`, `/K-Culture`
  - 관리자: `/K-CrossWord/admin/`

### 기술 스택
- **Spring Boot**: 8080 포트 통합 서비스
- **PostgreSQL**: 기존 데이터베이스 유지
- **Apache**: 리버스 프록시 (80 → 8080)
- **Redis**: 세션 관리
- **Thymeleaf**: 템플릿 엔진

### 2. 데이터베이스 스키마 호환성 해결

#### 2.1 엔티티 타입 수정
**문제**: JPA 엔티티의 `Long` 타입과 데이터베이스의 `integer` 타입 불일치

**해결**: 엔티티 타입을 데이터베이스 스키마에 맞게 수정
```java
// ThemeDailyPuzzle.java
private Integer id; // Long → Integer로 변경

// UserPuzzleCompletion.java  
private Integer id; // Long → Integer로 변경
```

#### 2.2 데이터베이스 스키마 유지
- 기존 PostgreSQL 스키마 그대로 유지
- 데이터베이스 수정 없이 엔티티만 조정

### 3. Spring Security 설정

#### 3.1 인증 규칙 설정
```java
.authorizeHttpRequests(authz -> authz
    .requestMatchers("/", "/index.html", "/puzzle", "/css/**", "/js/**", "/images/**", "/webjars/**", "/favicon.ico").permitAll()
    .requestMatchers("/Korean/**", "/K-pop/**", "/K-movie/**", "/K-Drama/**", "/K-Culture/**").permitAll()
    .requestMatchers("/board/**").permitAll()
    .requestMatchers("/api/puzzle/**", "/api/auth/**").permitAll()
    .requestMatchers("/K-CrossWord/admin/**").authenticated()
    .anyRequest().permitAll()  // 임시로 모든 요청 허용
)
```

## 🐛 해결된 문제들

### 1. 컴파일 오류
- **문제**: Lombok 의존성 누락
- **해결**: `build.gradle`에 Lombok 의존성 추가
- **문제**: 엔티티 import 경로 오류
- **해결**: 패키지 경로 수정 (`com.example.crossword` → `com.example.board`)

### 2. 데이터베이스 스키마 불일치
- **문제**: JPA 엔티티의 `Long` 타입과 DB의 `integer` 타입 불일치
- **해결**: 엔티티 타입을 `Integer`로 수정

### 3. Spring Boot 애플리케이션 시작 실패
- **문제**: Gemini API 설정 누락으로 인한 서비스 시작 실패
- **해결**: Gemini API 의존 서비스들을 임시 비활성화

### 4. 인증 API 404 오류
- **문제**: `/api/auth/status` 엔드포인트 누락
- **해결**: `AuthController`에 인증 상태 확인 API 추가

### 5. Spring Security 인증 문제
- **문제**: `/` 경로가 인증이 필요하다고 판단
- **해결**: SecurityConfig에서 `/` 경로를 명시적으로 `permitAll()` 설정
