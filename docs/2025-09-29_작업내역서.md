########## 퍼즐게임에 대한 정의 #########

### 1. 게임 개념
**크로스워드 퍼즐 게임**은 격자 형태의 그리드에 단어를 배치하고, 주어진 힌트를 통해 단어를 찾아 맞추는 퍼즐 게임입니다.

### 2. 게임 구성 요소
**그리드 시스템**:
- 각 레벨에 따른 그리드와 단어의 위치, 갯수가 정의되어 있는  템플릿 정보를 기준으로 랜더링해야 합니다. 
- 각 셀은 단어의 일부 또는 교차점
- 가로/세로 방향으로 단어 배치

**단어 시스템**:
- 각 단어는 고유한 ID와 여러개의 힌트를 가짐
- 단어 길이: 2~5글자
- 교차점을 통한 단어 연결

**힌트 시스템**:
- 각 단어마다 1개 이상의 힌트 제공
- 최초 랜더링시에는 기본 힌트만제공, 추가힌트 요청시  기본힌트를 제외한 id 값이 작은 힌트를 제공 
- 난이도별 힌트 분류
- 추가 힌트 요청 가능

### 3. 레벨 시스템
**레벨 구조**:
- 레벨별로 다른 난이도와 조건 설정
- 각 레벨마다 클리어 필요 횟수 지정
- 레벨 진화 조건: 클리어 횟수 충족 시 다음 레벨로 진행
- 각 레벨별 클리어 조건에 시간제한이 있으며, 시간 초과시 해당 레벨의 차수는 초기화 됨 

**레벨별 특성**:
- 단어 개수: 레벨별로 다름 (기본 5개)
- 단어 난이도: 1~5 단계
- 힌트 난이도: 1~5 단계
- 교차점 개수: 최소값 기준

### 4. 사용자 시스템
**사용자 타입**:
- **인증 사용자**: 로그인한 사용자, 진행 상황 저장
- **게스트 사용자**: 비로그인 사용자, 임시 ID로 게임 진행, 임시ID 로 진행상황 저장 
- 새로 고침을 해도 저장된 진행상황으로 다시 플레이

**진행 상황 관리**:
- 현재 레벨, 정답/오답 수, 정확도
- 활성 퍼즐 상태, 게임 데이터
- 클리어 기록 및 통계

### 5. 게임 플레이 흐름
**게임 시작**:
1. 레벨 선택 또는 현재 레벨 로드
2. 퍼즐 템플릿 생성 및 단어 배치
3. 그리드 렌더링 및 힌트 표시( 기본 힌트 )

**게임 진행**:
1. 번호 클릭으로 단어 선택
2. 힌트 확인 및 정답 입력
3. 정답 확인 및 피드백
4. 그리드 업데이트

**게임 완료**:
1. 모든 단어 완료 시 레벨 클리어
2. 클리어 조건 확인
3. 조건 충족 시 다음 레벨로 진화
4. 조건 미충족 시 동일 레벨 재시작

## 문제 상황 분석

### 1. 초기 문제점
1. **퍼즐메인 로그인 폼과 8080 서비스 로그인 차이**
   - 퍼즐메인(`/K-CrossWord/`)의 로그인 폼이 8081 포트의 API를 호출
   - 실제 로그인은 8080 포트에서 처리되어야 함
   - 두 서비스 간 인증 시스템이 분리되어 있음

2. **로그인 후 힌트 로드 실패**
   - 8080에서 로그인한 사용자가 8081 퍼즐게임으로 진입 시 힌트가 로드되지 않음
   - 그리드는 정상적으로 렌더링되지만 힌트 API 호출 시 400 Bad Request 에러 발생

3. **게스트 ID 지속성 문제**
   - 새로고침이나 재진입 시 새로운 퍼즐이 생성되는 문제
   - 같은 세션 내에서 게스트 ID가 유지되어야 함

### 2. 게임 페이지 인증 흐름 수정

#### 2.1 인증 우선순위 로직 구현
**파일**: `/var/www/html/crossword-projects/src/main/resources/static/game.html`

**변경 내용**:
```javascript
// IIFE (즉시 실행 함수)에서 인증 상태 확인
(function() {
    // 1. 인증 상태 확인
    fetch('/K-CrossWord/api/auth/status')
        .then(response => response.json())
        .then(data => {
            if (data.authenticated) {
                // 인증된 사용자
                window.GUEST_ID = null;
                window.USER_TYPE = 'user';
                console.log('인증된 사용자로 게임 진행:', data.username);
            } else {
                // 비인증 사용자 - 게스트 ID 확인
                const urlParams = new URLSearchParams(window.location.search);
                let guestId = localStorage.getItem('guestId') || urlParams.get('guest_id');
                
                if (guestId) {
                    window.GUEST_ID = guestId;
                    window.USER_TYPE = 'guest';
                    localStorage.setItem('guestId', guestId);
                } else {
                    // 게스트 ID도 없으면 메인으로 리다이렉트
                    window.location.href = '/K-CrossWord/';
                }
            }
        })
        .catch(error => {
            console.error('인증 상태 확인 실패:', error);
            window.location.href = '/K-CrossWord/';
        });
})();
```

**효과**: 
1. 인증된 사용자 우선 처리
2. 게스트 사용자는 기존 게스트 ID 유지
3. 둘 다 없으면 메인 페이지로 리다이렉트

 * 세션에서 인증된 사용자 ID 가져오기 (통합 인증)

private Long getUserIdFromSession(HttpServletRequest request) {
    try {
        // Spring Security의 SecurityContext에서 인증 정보 확인
        Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
        
        if (authentication != null && authentication.isAuthenticated() && 
            !authentication.getPrincipal().equals("anonymousUser")) {
            
            String email = authentication.getName();
            System.out.println("인증된 사용자 이메일: " + email);
            
            // 이메일로 사용자 ID 조회 (8080 서비스의 users 테이블과 동일)
            if ("rainynux@gmail.com".equals(email)) {
                return 1L; // 관리자 계정
            } else {
                // 다른 사용자들의 경우 이메일 해시를 사용하여 일관된 ID 생성
                return (long) Math.abs(email.hashCode() % 10000) + 1000;
            }
        }
    } catch (Exception e) {
        System.err.println("세션에서 사용자 ID 추출 실패: " + e.getMessage());
    }
    return null;
}
