## 1. 수동 퍼즐 전용 게임 페이지

### 신규 파일
- `crossword-projects/.../static/manual-game.html`

### 내용
- 테마 게임(`theme-game.html`)과 동일한 UI/UX를 기반으로 수동 퍼즐 전용 게임 페이지를 생성.
- API 호출 경로를 `MANUAL_API = '/K-CrossWord/admin/api/manual-puzzle'`로 분리하여 독립적인 도메인에서 동작.
- `WebController`에 `/manual-game` 라우트 추가. `/game?puzzleId=xxx` 접근 시 자동으로 `manual-game.html`로 라우팅.

## 2. 배지(번호) 매핑 수정

### 문제
- 수동 퍼즐 생성 화면에서 사용자가 지정한 배지 번호(`displayNum`)가 게임 화면의 `word_id`와 불일치.

### 수정 내용

#### 프론트엔드 (`admin/manual-puzzle/index.html`)
- `publishGame()`: `wordOrder` 생성 시 `display_num`을 포함하여 서버로 전송.
- `filledWithNum` 배열을 `displayNum` 기준으로 정렬 후 전송.

#### 백엔드 (`ManualPuzzleController.java`)
- `publishGame` 메서드: `item.get("display_num")`을 읽어 `slotNumber`로 사용. null이면 자동 증가값 사용.

#### 수정 화면 로드 시 번호 복원
- `loadExistingTemplate()`: 저장된 `wordPositions`(사용자 지정 번호)와 `analyzeGrid()` 결과를 **좌표 비교**하여 `wordNumberMap`을 먼저 복원한 뒤 `updateAnalysis()` 호출.
- 이를 통해 STEP 1 드롭다운과 STEP 2 슬롯 목록의 배지 번호가 저장 당시와 동일하게 표시.

## 3. 게임 상태 유지 (localStorage)

### 구현
- `manual-game.html`에서 `localStorage`를 사용하여 게임 진행 상태를 저장.
- 저장 항목: `answeredWords`, `answeredWordsData`, `hintsUsed`, `wrongAnswers`, `gameStartTime`
- 새로고침 시 기존 답변이 Wordle 스타일 플립 애니메이션과 함께 복원.
- 퍼즐 완료 시 `clearGameState()`로 해당 퍼즐의 상태 삭제.

## 4. 수동 퍼즐 힌트 API

### 신규 엔드포인트
```
GET /admin/api/manual-puzzle/hints
  ?word_id={pz_word_id}
  &base_hint_id={id}        (선택)
  &is_additional_hint={bool} (선택)
```

### 동작
- `pz_word_id`로 `pz_hints` 테이블에서 힌트 조회.
- `is_additional_hint=true`이면 `base_hint_id` 이후의 다음 힌트를 반환.
- 기본 힌트와 추가 힌트를 모두 지원.

## 7. 수동 퍼즐 목록 페이지

### 신규 파일
- `crossword-projects/.../static/admin/manual-puzzle/list.html`

### 백엔드 API

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `GET` | `/admin/api/manual-puzzle/list?filter=all\|active` | 수동 퍼즐 목록 (통계 포함) |
| `POST` | `/admin/api/manual-puzzle/{id}/deactivate` | 퍼즐 비활성화 (`isActive=false`) |
| `POST` | `/admin/api/manual-puzzle/{id}/activate` | 퍼즐 활성화 복원 |

### Repository 추가
- `ThemeDailyPuzzleRepository`: `findAllManualPuzzles()`, `findActiveManualPuzzles()` — theme이 `MANUAL-%`로 시작하는 레코드 조회.

### 목록 화면 기능
- 테이블 컬럼: #, 제목, Puzzle ID, 단어 수, **통계**(참여자 수, 평균 완료시간, 최고 기록), 생성일, 상태, 관리
- 필터: 전체 / 활성만
- 버튼: [게임] (새 탭), [수정] (편집 페이지 이동), [삭제] / [복원]
- `WebController`에 `/admin/manual-puzzle/list` 라우트 추가.

## 9. 수정(편집) 기능 — 기존 템플릿 로드

### 동작
수동 퍼즐 목록에서 [수정] 클릭 → `/K-CrossWord/admin/manual-puzzle?templateId={id}`로 이동.

### 복원 순서
1. `GET /admin/api/manual-puzzle/template/{id}` API로 템플릿 데이터 로드
2. **그리드 패턴** 복원 (`gridPattern` → `currentGrid`)
3. **wordNumberMap 복원**: 저장된 `wordPositions`(사용자 지정 번호)와 `analyzeGrid()` 결과를 좌표 비교하여 매핑
4. `updateAnalysis()` 호출 → STEP 1 드롭다운에 올바른 번호 반영
5. **레벨** 자동 선택
6. **단어/힌트 복원**: `extractedWordsSnapshot.word_order`에서 좌표 매칭으로 슬롯에 배치
7. 이후 일부 단어 변경, 힌트 수정 후 재발행 가능
