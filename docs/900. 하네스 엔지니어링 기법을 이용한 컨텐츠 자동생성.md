# 하네스 엔지니어링 기법을 이용한 콘텐츠 자동생성

> 원본 구조: HARNESS_SYSTEM_ARCHITECTURE

## 1. 개요 (Overview)

| 항목 | 내용 |
|------|------|
| **명칭** | 하네스 (Harness) |
| **목적** | 숏폼 트렌드 역설계 기반 콘텐츠 자동 생성 및 배포 |
| **핵심 로직** | 트렌드 DNA 추출 → 데이터 매핑 → 에셋 생성 → FFmpeg 동적 렌더링 |

- **작동 원리·규칙·알고리즘·데이터 모델·UI/UX**: [`100.퍼즐게임의 작동원리와 서비스 개요.md`](./100.퍼즐게임의 작동원리와 서비스 개요.md) — K-CrossWord(한글 십자낱말), 템플릿·단어·테마 일일 퍼즐 등
- **화면 참고 이미지**: 워크스페이스 `harness-factory/ref/` (게임·관리자 스크린샷)

## 2. 시스템 아키텍처 (Engine Components)

### Engine-0: 역설계 (Reverse Engineering)

- **영상 구조 분석**: 자막 Safe Zone, 폰트 크기, HEX 컬러 추출
- **오디오 분석**: BGM BPM 측정 및 Peak 지점 데이터화 (자막 Sync용)
- **패턴 분류**: 도입 1.5초 후킹 문구 카테고리화

### Engine-1: 데이터 정합 (Contextual Mapping)

- **페르소나·콘텐츠 매칭**: Engine-0 트렌드 메타와 **퍼즐 도메인**(단어·테마·난이도 등, 위 `100` 문서)을 연결
- **v1**: 트렌드 → 일일 테마 코드 매핑 — [`901.Engine-1_스펙_초안.md`](./901.Engine-1_스펙_초안.md)
- **스크립트 튜닝** (후속): 원본 밈의 음절 수 및 운율 유지 (변주 로직)

### Engine-2: 에셋 미믹 (Virtual Studio)

- **비주얼**: 스타일 트랜스퍼를 통한 저작권 회피 및 독창적 스타일 생성
- **오디오**: 상황별(분노, 희열 등) 감정 파라미터가 적용된 TTS 생성

### Engine-3: 동적 렌더링 (Dynamic Producer)

- **엔진**: FFmpeg 기반 코드 렌더링
- **특수효과**: 수학적 함수(sin, cos) 기반 자막 바운스 및 트랜지션
- **최적화**: OCI 자원 효율을 위한 가변 비트레이트(VBR) 인코딩

### Engine-4: 피드백 (Smart Feedback)

- **분석**: A/B 테스트 및 댓글 감성 분석
- **학습**: 성과 지표에 따른 Engine-0 가중치 피드백

## 3. 현행 단계 성과 (Current Status)

### 환경 및 인프라

- OCI(Ubuntu) 내 Playwright 구동 환경 구축 완료
- Stealth 모드 및 Cookie 주입을 통한 봇 감지 우회 성공

### 데이터 수집 (Engine-0 기반)

- 쇼츠 피드 무한 스크롤 및 실시간 URL 100건 수집 로직 구현
- SameSite 쿠키 에러 및 데이터 센터 전용 IP 차단 이슈 해결
- YouTube: 검색·필터 기반 `scanner.py` → `engine0_shorts.json`
- Instagram: `reels_scanner.py` → `engine0_reels.json`

### Phase 1: PostgreSQL 자산화

- DDL: `harness-factory/engine-0-scanner/sql/001_engine0_trends.sql`
- 적재: `ingest_to_postgres.py` — `engine0_shorts.json` / `engine0_reels.json`을 `engine0_trends`에 UPSERT (`url` 기준)
- 연결: `crossword/.env` 의 `NATUS_DB_HOST`·`PORT`·`USER`·`NAME`·`PASSWORD`(Spring 과 동일) — 스크립트가 `db_env.py`로 읽음. 필요 시 `DATABASE_URL` 단독 사용 가능
- 의존성: `pip install -r engine-0-scanner/requirements-db.txt`

### Phase 2: AI 메타데이터 (Gemini)

- 스크립트: `enrich_gemini.py` — `category`, `summary_features`, `hooking_score` 보강 (미입력 행만, `--force`로 재생성)
- **API 키**: `crossword/.env` 의 `GEMINI_API_KEY` 를 자동 로드 (경로 변경 시 `HARNESS_DOTENV_PATH`). 별도 `export` 불필요.
- 선택: `GEMINI_MODEL`(기본 `gemini-flash-latest`, 구 `gemini-2.0-flash` 는 신규 API 키에서 사용 불가)
- 의존성: `pip install -r engine-0-scanner/requirements-ai.txt`

### Phase 3: 일일 자동화

- `daily_engine0_run.sh` + `report_engine0_summary.py` + `crontab.example` (아래 로드맵과 동일 설명)

### Engine-1 v1: 테마 정합

- 스펙: [`901.Engine-1_스펙_초안.md`](./901.Engine-1_스펙_초안.md)
