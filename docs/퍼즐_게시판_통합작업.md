# 퍼즐, 게시판 통합작업

## 📋 작업 개요

### 요청사항
- **목표**: 8080(게시판)과 8081(퍼즐게임) Spring Boot 애플리케이션을 단일 애플리케이션(8080)으로 통합
- **도메인**: `natus250601.viewdns.net`
- **라우팅 구조**:
  - 메인 화면: `/` (기존 퍼즐게임 메인과 동일)
  - 게시판: `/board`
  - 퍼즐게임: `/Korean/game`, `/K-pop`, `/K-movie`, `/K-Drama`, `/K-Culture`
  - 관리자: `/K-CrossWord/admin/`

### 기술 스택
- **Spring Boot**: 8080 포트 통합 서비스
- **PostgreSQL**: 기존 데이터베이스 유지
- **Apache**: 리버스 프록시 (80 → 8080)
- **Redis**: 세션 관리
- **Thymeleaf**: 템플릿 엔진

```javascript
// 인증 상태 확인
function checkAuthStatus() {
    fetch('/api/auth/status', {
        method: 'GET',
        credentials: 'include'
    })
    .then(response => response.json())
    .then(data => {
        if (data.authenticated && data.userType === 'user') {
            // 인증된 사용자 - 게임시작 버튼 표시
            userButtons.style.setProperty('display', 'flex', 'important');
        } else {
            // 비인증 사용자 - 기본 버튼 표시
            guestButtons.style.setProperty('display', 'flex', 'important');
        }
    })
    .catch(error => {
        // 오류 발생 시 비인증 상태로 처리
        guestButtons.style.setProperty('display', 'flex', 'important');
    });
}
```

## 🐛 해결된 문제들

### 1. 컴파일 오류
- **문제**: Lombok 의존성 누락
- **해결**: `build.gradle`에 Lombok 의존성 추가
- **문제**: 엔티티 import 경로 오류
- **해결**: 패키지 경로 수정 (`com.example.crossword` → `com.example.board`)

### 2. 라우팅 충돌
- **문제**: `@GetMapping("/")` 중복 매핑
- **해결**: `HomeController`와 `PuzzleController` 라우팅 분리

### 3. API 응답 구조 불일치
- **문제**: JavaScript가 기대하는 `userType` 필드 누락
- **해결**: `AuthController` 응답에 `userType` 필드 추가

### 4. CORS 문제
- **문제**: 브라우저에서 API 호출 시 CORS 오류
- **해결**: `@CrossOrigin(origins = "*")` 어노테이션 추가

### 5. Spring Boot 애플리케이션 시작 실패
- **문제**: 애플리케이션 시작 후 즉시 종료
- **해결**: `nohup` 명령어로 백그라운드 실행 보장

### 작업 개요
- 관리자 페이지에 개발서버와 운영서버 간 데이터베이스 동기화 기능 구현
- 테이블 목록에 테이블 코멘트(설명) 표시 기능 추가
- 테이블 동기화 시 코멘트도 함께 동기화하는 기능 구현
- 32개 테이블에 코멘트 추가

### 주요 구현 사항

#### 1. 데이터베이스 동기화 페이지 구현
**백엔드 구현:**
- `DatabaseSyncService.java`: 데이터베이스 동기화 서비스 클래스 생성
  - `getLocalTables()`: 로컬(개발) 서버 테이블 목록 조회
  - `getRemoteTables()`: 원격(운영) 서버 테이블 목록 조회
  - `syncTable()`: 테이블 동기화 메인 메서드
  - `getTableInfo()`: 테이블 정보 조회 (로우 개수, 최종 갱신일, 코멘트)
  - `createTableIfNotExists()`: 원격 서버에 테이블이 없으면 로컬 스키마로 생성
  - `getCreateTableStatement()`: PostgreSQL 스키마 기반 CREATE TABLE 문 동적 생성
  - `syncTableComment()`: 테이블 코멘트 동기화 메서드
  - `getTableData()`: 테이블 데이터 조회
  - `insertDataToRemote()`: 원격 서버에 데이터 일괄 삽입

**API 엔드포인트:**
- `GET /admin/api/db-sync/local-tables`: 개발 서버 테이블 목록 조회
- `GET /admin/api/db-sync/remote-tables`: 운영 서버 테이블 목록 조회
- `POST /admin/api/db-sync/sync/{tableName}`: 테이블 동기화 실행

**프론트엔드 구현:**
- `static/admin/db-sync/index.html`: 데이터베이스 동기화 관리 페이지
  - 좌측: 개발 서버 테이블 목록
  - 우측: 운영 서버 테이블 목록
  - 테이블 선택 후 동기화 버튼으로 실행
  - 테이블명, 로우 개수, 최종 갱신일, 테이블 코멘트 표시

#### 2. 테이블 코멘트 표시 기능 추가
**구현 내용:**
- 개발 서버 테이블 목록에 테이블 코멘트 표시
- 표시 형식: `테이블명 (코멘트)` 또는 코멘트가 없으면 `테이블명`만 표시
- `getTableInfo()` 메서드에서 `obj_description()` 함수로 코멘트 조회

**데이터베이스 코멘트 추가:**
- 32개 테이블에 코멘트 추가 완료
- 주요 테이블 코멘트:
  - `pz_words`: 퍼즐 단어 테이블 - 십자낱말 퀴즈에 사용되는 단어 저장
  - `pz_hints`: 퍼즐 힌트 테이블 - 퍼즐 단어에 대한 힌트 저장
  - `puzzle_levels`: 퍼즐 레벨 테이블 - 레벨별 난이도, 단어 수, 교차점 수, 시간 제한 등 설정
  - `boards`: 게시판 테이블 - 게시글 저장
  - `users`: 사용자 테이블 - 시스템 사용자 정보 저장
  - 기타 27개 테이블 코멘트 추가

#### 3. 인코딩 문제 해결
**문제:**
- 테이블 코멘트가 깨져서 표시됨 (예: `⑥ 蹂`)
- PostgreSQL JDBC 연결 시 UTF-8 인코딩 미지정

**해결 방법:**
- JDBC URL에 인코딩 파라미터 추가: `characterEncoding=utf8&useUnicode=true&stringtype=unspecified`
- 연결 생성 시 `SET client_encoding TO 'UTF8'` 명령 실행
- `application.properties`에 인코딩 설정 추가

#### 4. 테이블 동기화 시 코멘트 동기화 기능
**구현 내용:**
- `syncTableComment()` 메서드 추가
- 동기화 프로세스:
  1. 로컬 DB에서 테이블 코멘트 조회
  2. 원격 DB에 동일한 코멘트 적용
  3. 코멘트가 없으면 원격에서 제거
- SQL 인젝션 방지: 작은따옴표 이스케이프 처리 (`'` → `''`)
- 코멘트 동기화 실패해도 데이터 동기화는 계속 진행

#### 5. 동적 테이블 생성 기능
**구현 내용:**
- 원격 서버에 테이블이 없으면 자동 생성
- `getCreateTableStatement()`: PostgreSQL 스키마 기반 CREATE TABLE 문 동적 생성
  - 컬럼 정보 (이름, 데이터 타입, 길이, NULL 제약, 기본값)
  - PRIMARY KEY 제약조건
  - 시퀀스 생성 및 연결 (SERIAL 타입)
- PRIMARY KEY 조회 시 `key_column_usage` 사용 (이전 `constraint_column_usage` 오류 수정)

### 해결된 문제들

#### PRIMARY KEY 조회 오류
**문제**: `constraint_column_usage`에는 `ordinal_position` 컬럼이 없음
**해결**: `key_column_usage` 사용으로 변경

#### 인코딩 문제
**문제**: 테이블 코멘트가 깨져서 표시됨
**해결**: JDBC URL 인코딩 파라미터 추가 및 연결 시 UTF-8 설정

#### COMMENT ON TABLE 플레이스홀더 미지원
**문제**: PostgreSQL `COMMENT ON TABLE`은 PreparedStatement 플레이스홀더 미지원
**해결**: Statement 사용 및 SQL 인젝션 방지를 위한 이스케이프 처리

#### 1. 정제완료 통계 수정 (단어 관리, 퍼즐 관리 메인 화면)

**문제점**: 
- 정제완료 카드가 힌트가 있는 단어 수(`getWordsWithHintsCount()`)를 표시하고 있었음
- 실제로는 `conf_yn = 'Y'`인 단어 수를 표시해야 함
- 현재 실제 정제완료 단어는 225개인데 37,960개로 표시되고 있었음

**해결 방법**:
1. `PzWordRepository`에 `countByConfYn(String confYn)` 메서드 추가
   - Spring Data JPA 메서드 네이밍 규칙으로 자동 생성되는 쿼리: `SELECT COUNT(*) FROM pz_words WHERE conf_yn = ?`

2. `PzWordService`에 `getRefinedWordsCount()` 메서드 추가
   ```java
   public long getRefinedWordsCount() {
       return pzWordRepository.countByConfYn("Y");
   }
   ```

3. `AdminApiController`의 두 통계 API 엔드포인트 수정
   - `/admin/api/stats`: 시스템 전체 통계
   - `/admin/api/words/stats`: 단어 통계
   - 두 엔드포인트 모두 `getWordsWithHintsCount()` 대신 `getRefinedWordsCount()` 사용

**영향 받는 화면**:
- 퍼즐 관리 메인 화면 (`/admin/`)
- 단어 관리 화면 (`/admin/words`)

**영향 받는 화면**:
- AI 힌트 관리 화면 (`/admin/hint-generator`)

**쿼리**:

#### 3. Map 불변성 문제 해결
- **문제**: `ThemePuzzleApiController.getTodayPuzzle()`에서 `Map.of()`로 생성한 불변 Map에 `put()` 시도로 인한 `UnsupportedOperationException`
  ```

  at java.util.ImmutableCollections$AbstractImmutableMap.put
  ```
- **원인**: `Map.of()`는 불변(immutable) Map을 반환하므로 수정 불가
- **해결**: `HashMap`으로 변환하여 수정 가능하게 처리

  ```java
  // 응답에 테마와 날짜 정보 추가 (HashMap으로 변환하여 수정 가능하게)
  Map<String, Object> response = new HashMap<>(puzzleData);

  return ResponseEntity.ok(response);
  ```
