### 1. 라디오 버튼 선택에 따른 입력 필드 Hide/Show 기능 수정

#### 문제점
- 라디오 버튼("메뉴별" / "일별") 선택 시 입력 필드가 제대로 숨겨지지 않거나 표시되지 않는 문제
- Bootstrap의 `d-flex` 클래스가 JavaScript로 설정한 `display` 속성을 덮어쓰는 CSS 우선순위 문제

#### 해결 방법
- **HTML 수정** (`index.html`):
  - `yearMonthInputs` div의 초기 스타일을 `display: none !important;`로 설정
  - `dateRangeButtons` div의 초기 스타일을 `display: flex !important;`로 설정
  
- **JavaScript 수정** (`toggleDateInputs()` 함수):
  - `style.display` 직접 설정 대신 `setAttribute('style', 'display: flex !important;')` 방식 사용
  - CSS 우선순위 문제를 해결하기 위해 `!important` 플래그 추가
  - 디버깅을 위한 console.log 제거 (최종 버전)

#### 수정 파일
- `/var/www/html/java-projects/src/main/resources/static/project/kalisco/crawling/saleSituation/index.html`

#### 수정 내용 상세
```javascript
function toggleDateInputs() {
    const saleType = document.querySelector('input[name="saleType"]:checked').value;
    const dateRangeInputs = document.getElementById('dateRangeInputs');
    const yearMonthInputs = document.getElementById('yearMonthInputs');
    const dateRangeButtons = document.getElementById('dateRangeButtons');
    
    if (saleType === 'menu') {
        // 메뉴별: 시작일/종료일 입력 필드와 날짜 버튼 표시, 연도/월 숨김
        if (dateRangeInputs) {
            dateRangeInputs.setAttribute('style', 'display: flex !important;');
        }
        if (yearMonthInputs) {
            yearMonthInputs.setAttribute('style', 'display: none !important;');
        }
        if (dateRangeButtons) {
            dateRangeButtons.setAttribute('style', 'display: flex !important;');
        }
    } else {
        // 일별: 연도/월만 표시, 시작일/종료일 입력 필드와 날짜 버튼 숨김
        if (dateRangeInputs) {
            dateRangeInputs.setAttribute('style', 'display: none !important;');
        }
        if (yearMonthInputs) {
            yearMonthInputs.setAttribute('style', 'display: flex !important;');
        }
        if (dateRangeButtons) {
            dateRangeButtons.setAttribute('style', 'display: none !important;');
        }
    }
}
```

### 2. 일별 매출 데이터 헤더 순서 문제 해결

#### 문제점
- 일별 매출 원본 페이지는 "일자, 매출액(원), 영수건수, 고객수" 순서로 표시
- 크롤링된 데이터가 다른 순서로 표시되는 문제
- `HashMap` 사용으로 인한 순서 보장 불가

#### 해결 방법
- **백엔드 수정** (`OurHomeScraperService.java`):
  - `HashMap` → `LinkedHashMap`으로 변경하여 삽입 순서 보장
  - 데이터 행을 Map에 넣을 때 헤더 순서대로 유지
  
- **프론트엔드 수정** (`index.html`):
  - `displayData()` 함수가 API 응답의 `headers` 배열을 받도록 수정
  - 헤더 순서를 API에서 받은 순서대로 사용
  - 원본 페이지 순서 유지

#### 수정 파일
- `/var/www/html/java-projects/src/main/java/com/example/board/service/OurHomeScraperService.java`
- `/var/www/html/java-projects/src/main/resources/static/project/kalisco/crawling/saleSituation/index.html`

#### 수정 내용 상세

**백엔드 (OurHomeScraperService.java)**:
```java
import java.util.LinkedHashMap;  // 추가

// 데이터 행 추출 시 LinkedHashMap 사용

int maxCols = Math.min(headers.size(), cells.size());

}
```

## 참고 사항
- Bootstrap의 `d-flex` 클래스는 `display: flex`를 강제하므로, JavaScript로 동적 제어 시 `!important` 플래그가 필요함
- JavaScript의 `Object.keys()`는 삽입 순서를 보장하지만, JSON 직렬화/역직렬화 과정에서 순서가 바뀔 수 있으므로 API에서 명시적으로 헤더 순서를 전달하는 것이 안전함
- `LinkedHashMap`은 삽입 순서를 보장하므로, 헤더 순서가 중요한 경우 반드시 사용해야 함
