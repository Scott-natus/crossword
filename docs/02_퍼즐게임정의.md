# 퍼즐게임 시스템 정의

## 1. 게임 개념

**크로스워드 퍼즐 게임**은 격자 형태의 그리드에 단어를 배치하고, 주어진 힌트를 통해 단어를 찾아 맞추는 퍼즐 게임입니다.

## 2. 게임 구성 요소

### 그리드 시스템
- 각 레벨에 따른 그리드와 단어의 위치, 갯수가 정의되어 있는 템플릿 정보를 기준으로 랜더링해야 합니다.
- 각 셀은 단어의 일부 또는 교차점
- 가로/세로 방향으로 단어 배치

### 단어 시스템
- 각 단어는 고유한 ID와 여러개의 힌트를 가짐
- 단어 길이: 2~5글자
- 교차점을 통한 단어 연결

### 힌트 시스템
- 각 단어마다 1개 이상의 힌트 제공
- 최초 랜더링시에는 기본 힌트만제공, 추가힌트 요청시 기본힌트를 제외한 id 값이 작은 힌트를 제공
- 난이도별 힌트 분류
- 추가 힌트 요청 가능

## 3. 레벨 시스템

### 레벨 구조
- 레벨별로 다른 난이도와 조건 설정
- 각 레벨마다 클리어 필요 횟수 지정
- 레벨 진화 조건: 클리어 횟수 충족 시 다음 레벨로 진행
- 각 레벨별 클리어 조건에 시간제한이 있으며, 시간 초과시 해당 레벨의 차수는 초기화 됨

### 레벨별 특성
- 단어 개수: 레벨별로 다름 (기본 5개)
- 단어 난이도: 1~5 단계
- 힌트 난이도: 1~5 단계
- 교차점 개수: 최소값 기준

## 4. 사용자 시스템

### 사용자 타입
- **인증 사용자**: 로그인한 사용자, 진행 상황 저장
- **게스트 사용자**: 비로그인 사용자, 임시 ID로 게임 진행, 임시ID 로 진행상황 저장
- 새로 고침을 해도 저장된 진행상황으로 다시 플레이

### 인증/게스트 플레이 테이블 구조

#### users 테이블
```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY,
    email VARCHAR(255),
    password VARCHAR(255),
    name VARCHAR(255),
    is_guest BOOLEAN DEFAULT FALSE,
    guest_id UUID,
    is_admin BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

**컬럼 설명**:
- `id`: 사용자 고유 ID (인증 사용자: 실제 ID, 게스트 사용자: 임시 생성된 숫자 ID)
- `email`: 이메일 주소 (인증 사용자만)
- `password`: 비밀번호 (인증 사용자만)
- `name`: 사용자 이름
- `is_guest`: 게스트 사용자 여부 (TRUE: 게스트, FALSE: 인증 사용자)
- `guest_id`: 게스트 고유 UUID (게스트 사용자만)
- `is_admin`: 관리자 권한 여부
- `created_at`, `updated_at`: 생성/수정 시간

#### user_puzzle_games 테이블
```sql
CREATE TABLE user_puzzle_games (
    id BIGINT PRIMARY KEY,
    user_id BIGINT,                    -- users.id 참조
    guest_id UUID,                     -- users.guest_id 참조 (게스트용)
    current_level INTEGER DEFAULT 1,
    first_attempt_at TIMESTAMP,
    total_play_time INTEGER DEFAULT 0,
    accuracy_rate DECIMAL(5,2) DEFAULT 0.00,
    total_correct_answers INTEGER DEFAULT 0,
    total_wrong_answers INTEGER DEFAULT 0,
    current_level_correct_answers INTEGER DEFAULT 0,
    current_level_wrong_answers INTEGER DEFAULT 0,
    ranking INTEGER DEFAULT 0,
    last_played_at TIMESTAMP,
    is_active BOOLEAN DEFAULT TRUE,
    current_puzzle_data JSON,          -- 현재 퍼즐 데이터 (그리드, 단어, 힌트)
    current_game_state JSON,           -- 현재 게임 상태 (정답한 단어, 사용한 힌트 등)
    current_puzzle_started_at TIMESTAMP,
    has_active_puzzle BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);
```

**컬럼 설명**:
- `user_id`: users 테이블의 id 참조 (인증/게스트 모두)
- `guest_id`: 게스트 사용자의 UUID (게스트용, 인증 사용자는 NULL)
- `current_level`: 현재 진행 중인 레벨
- `current_puzzle_data`: 현재 퍼즐의 전체 데이터 (JSON 형태)
  - `extracted_words`: 추출된 단어 목록
  - `grid_pattern`: 그리드 패턴
  - `template_id`: 사용된 템플릿 ID
- `current_game_state`: 현재 게임 상태 (JSON 형태)
  - `answered_words`: 정답한 단어 목록
  - `hints_used`: 사용한 힌트 목록
  - `wrong_answers`: 오답 목록
- `has_active_puzzle`: 활성 퍼즐 존재 여부
- `is_active`: 게임 세션 활성 상태

### 게임 진행 프로세스

#### 1. 게임 진입 프로세스
```
1. 사용자 접속
   ↓
2. 인증 상태 확인
   ├─ 인증 사용자: 기존 user_id 사용
   └─ 게스트 사용자: guest_id 생성 또는 기존 guest_id 사용
   ↓
3. user_puzzle_games 테이블에서 활성 게임 조회
   ├─ 활성 게임 있음: 기존 게임 상태 복원
   └─ 활성 게임 없음: 새 게임 시작
   ↓
4. 퍼즐 데이터 로드 및 그리드 렌더링
```

#### 2. 게스트 사용자 ID 생성 규칙
```javascript
// 클라이언트에서 게스트 ID 생성
const guestId = `guest_${Date.now()}`;  // 예: "guest_1758994129146"

// 서버에서 users 테이블에 게스트 사용자 생성
const guestUser = {
    id: Long.valueOf(guestId.replace("guest_", "")),  // 1758994129146
    is_guest: true,
    guest_id: UUID.randomUUID(),
    created_at: LocalDateTime.now()
};
```

#### 3. 게임 상태 저장 프로세스
```
1. 정답 입력 시
   ↓
2. 서버에서 정답 검증
   ↓
3. 정답인 경우:
   ├─ current_game_state에 정답 단어 추가
   ├─ current_level_correct_answers 증가
   └─ 그리드에 정답 표시
   ↓
4. 오답인 경우:
   ├─ current_level_wrong_answers 증가
   └─ 오답 5회 초과 시 게임 종료 처리
   ↓
5. user_puzzle_games 테이블 업데이트
```

#### 4. 오답 초과 시 게임 종료 프로세스
```
1. 오답 5회 초과 감지
   ↓
2. 서버에서 정답 정보 추출 (current_puzzle_data에서)
   ↓
3. 클라이언트에 정답 정보 전송 (all_answers)
   ↓
4. 게임 상태 초기화:
   ├─ has_active_puzzle = false
   ├─ current_puzzle_data = null
   ├─ current_game_state = null
   └─ current_puzzle_started_at = null
   ↓
5. 클라이언트에서:
   ├─ 게임종료 모달 표시
   ├─ 정답들을 0.5초 간격으로 순차 애니메이션
   └─ 재도전/취소 선택
```

#### 5. 재도전 프로세스
```
1. 재도전 버튼 클릭
   ↓
2. 페이지 새로고침
   ↓
3. has_active_puzzle = false이므로 새 게임 시작
   ↓
4. 새로운 템플릿, 단어, 힌트 생성
   ↓
5. 게임 상태 초기화 후 새 퍼즐 시작
```

## 12. 레벨 클리어 조건 시스템

### 시스템 구조
**레벨 클리어 조건**:
- 각 레벨마다 정해진 횟수만큼 클리어해야 다음 레벨로 진화
- 단순히 모든 단어를 맞추는 것이 아니라, 동일 레벨을 반복 플레이해야 함
- `puzzle_levels.clear_condition` 컬럼에 클리어 필요 횟수 저장

**클리어 기록 관리**:
- `puzzle_game_records` 테이블에 클리어 기록 저장
- `game_status = 'completed'`인 레코드가 클리어 횟수로 계산
- 각 클리어마다 게임 데이터(점수, 시간, 힌트 사용량 등) 저장

### API 동작 로직
**레벨 완료 처리 (`complete-level` API)**:
1. 현재 퍼즐 완료 기록 저장
2. 레벨 클리어 조건 확인
3. 조건 미충족 시 400 에러 반환 (500 에러는 서버 문제)
4. 조건 충족 시 다음 레벨로 진화

**클리어 조건 확인 로직**:
```php
// 라라벨 로직
$clearCount = DB::table('puzzle_game_records')
    ->where('user_id', $userId)
    ->where('level_played', $currentLevel)
    ->where('game_status', 'completed')
    ->count();

$canAdvance = $clearCount >= $level->clear_condition;
```

### 데이터베이스 스키마
**puzzle_levels 테이블**:
- `clear_condition`: 레벨 클리어 필요 횟수 (기본값: 1)
- 예: 레벨 2는 `clear_condition = 5`

**puzzle_game_records 테이블**:
- `level_played`: 플레이한 레벨
- `game_status`: 게임 상태 ('completed'가 클리어로 인정)
- `words_found`, `total_words`: 클리어 조건 확인용
- `level_up`: 레벨 진화 여부
