# 퍼즐게임 시스템 정의

## 1. 게임 개념

**크로스워드 퍼즐 게임**은 격자 형태의 그리드에 단어를 배치하고, 주어진 힌트를 통해 단어를 찾아 맞추는 퍼즐 게임입니다.

## 2. 게임 구성 요소

### 그리드 시스템
- 각 레벨에 따른 그리드와 단어의 위치, 갯수가 정의되어 있는 템플릿 정보를 기준으로 랜더링해야 합니다.
- 각 셀은 단어의 일부 또는 교차점
- 가로/세로 방향으로 단어 배치

### 단어 시스템
- 각 단어는 고유한 ID와 여러개의 힌트를 가짐
- 단어 길이: 2~5글자
- 교차점을 통한 단어 연결

### 힌트 시스템
- 각 단어마다 1개 이상의 힌트 제공
- 최초 랜더링시에는 기본 힌트만제공, 추가힌트 요청시 기본힌트를 제외한 id 값이 작은 힌트를 제공
- 난이도별 힌트 분류
- 추가 힌트 요청 가능

## 3. 레벨 시스템

### 레벨 구조
- 레벨별로 다른 난이도와 조건 설정
- 각 레벨마다 클리어 필요 횟수 지정
- 레벨 진화 조건: 클리어 횟수 충족 시 다음 레벨로 진행
- 각 레벨별 클리어 조건에 시간제한이 있으며, 시간 초과시 해당 레벨의 차수는 초기화 됨

### 레벨별 특성
- 단어 개수: 레벨별로 다름 (기본 5개)
- 단어 난이도: 1~5 단계
- 힌트 난이도: 1~5 단계
- 교차점 개수: 최소값 기준

## 4. 사용자 시스템

### 사용자 타입
- **인증 사용자**: 로그인한 사용자, 진행 상황 저장
- **게스트 사용자**: 비로그인 사용자, 임시 ID로 게임 진행, 임시ID 로 진행상황 저장
- 새로 고침을 해도 저장된 진행상황으로 다시 플레이

### 진행 상황 관리
- 현재 레벨, 정답/오답 수, 정확도
- 활성 퍼즐 상태, 게임 데이터
- 클리어 기록 및 통계

## 5. 게임 플레이 흐름

### 게임 시작
1. 레벨 선택 또는 현재 레벨 로드
2. 퍼즐 템플릿 생성 및 단어 배치
3. 그리드 렌더링 및 힌트 표시( 기본 힌트 )

### 게임 진행
1. 번호 클릭으로 단어 선택
2. 힌트 확인 및 정답 입력
3. 정답 확인 및 피드백
4. 그리드 업데이트

### 게임 완료
1. 모든 단어 완료 시 레벨 클리어
2. 클리어 조건 확인
3. 조건 충족 시 다음 레벨로 진화
4. 조건 미충족 시 동일 레벨 재시작

## 7. 데이터베이스 구조

### 핵심 테이블
- `users`: 사용자 정보
- `user_puzzle_games`: 사용자 게임 진행 상황
- `puzzle_levels`: 레벨 설정 및 클리어 조건
- `puzzle_game_records`: 게임 클리어 기록
- `pz_words`: 퍼즐 단어 데이터
- `pz_hints`: 힌트 데이터

### 관계 구조

- 레벨 ↔ 클리어 기록 (1:N)
- 단어 ↔ 힌트 (1:N)
- 템플릿 ↔ 단어 (1:N)

## 12. 레벨 클리어 조건 시스템

### 시스템 구조
**레벨 클리어 조건**:
- 각 레벨마다 정해진 횟수만큼 클리어해야 다음 레벨로 진화
- 단순히 모든 단어를 맞추는 것이 아니라, 동일 레벨을 반복 플레이해야 함
- `puzzle_levels.clear_condition` 컬럼에 클리어 필요 횟수 저장

**클리어 기록 관리**:
- `puzzle_game_records` 테이블에 클리어 기록 저장
- `game_status = 'completed'`인 레코드가 클리어 횟수로 계산
- 각 클리어마다 게임 데이터(점수, 시간, 힌트 사용량 등) 저장

### API 동작 로직
**레벨 완료 처리 (`complete-level` API)**:
1. 현재 퍼즐 완료 기록 저장
2. 레벨 클리어 조건 확인
3. 조건 미충족 시 400 에러 반환 (500 에러는 서버 문제)
4. 조건 충족 시 다음 레벨로 진화

**클리어 조건 확인 로직**:
```php
// 라라벨 로직
$clearCount = DB::table('puzzle_game_records')
    ->where('user_id', $userId)
    ->where('level_played', $currentLevel)
    ->where('game_status', 'completed')
    ->count();

$canAdvance = $clearCount >= $level->clear_condition;
```

### 데이터베이스 스키마
**puzzle_levels 테이블**:
- `clear_condition`: 레벨 클리어 필요 횟수 (기본값: 1)
- 예: 레벨 2는 `clear_condition = 5`

**puzzle_game_records 테이블**:
- `level_played`: 플레이한 레벨
- `game_status`: 게임 상태 ('completed'가 클리어로 인정)
- `words_found`, `total_words`: 클리어 조건 확인용
- `level_up`: 레벨 진화 여부
