**우선순위**: 최고
**예상 소요시간**: 20일 (테마별 단어 수집 및 힌트 생성 시스템 추가로 3일 연장)

- **목표**: 테마별 퍼즐 및 다국어 힌트 지원을 위한 데이터베이스 구조 확장
- **작업 내용**:
  - `pz_words` 테이블에 `CAT2` 컬럼 추가 (K-POP, K-DRAMA, K-MOVIE, K-CULTURE, Korean)
  - `pz_words` 테이블에 `is_approved` 컬럼 추가 (관리자 승인 시스템)
  - `pz_hints` 테이블에 `language_code` 컬럼 추가 (다국어 힌트 지원)
  - `pz_hints` 테이블에 `is_approved` 컬럼 추가
  - `theme_daily_puzzles` 테이블 생성 (테마별 일일 퍼즐 관리)
  - `user_puzzle_completions` 테이블 생성 (사용자 완료 기록)
  - `newsletter_subscriptions` 테이블 생성 (뉴스레터 구독)

  - 힌트 품질 검증 및 승인 시스템
- **작업 내용**:
  - `ThemeWordCollectionService` 구현
  - `MultilingualHintGenerationService` 구현
  - Gemini API 다국어 프롬프트 최적화
  - 테마별 단어 수집 인터페이스 구현
  - 힌트 품질 검증 시스템

## 2025-10-10 작업 완료 내역

### 레벨 게이지 표시 문제 해결
**작업일**: 2025-10-10  
**작업자**: Cursor AI Assistant  
**상태**: ✅ 완료

#### 문제 상황
- 레벨 게이지가 현재 퍼즐의 정답 개수를 표시하는 문제
- 게이지가 레벨 클리어 횟수가 아닌 정답 개수로 업데이트됨
- `clear_count` 값이 올바르게 표시되지 않는 문제

#### 해결 과정
1. **문제 분석**:
   - `clear_count`가 API 응답의 최상위 레벨에 위치
   - `gameData.clear_count`로 접근 시도했으나 `gameData` 객체에 해당 필드 없음
   - 디버깅 로그를 통해 `response.clear_count: 1`은 정상, `clearCount: 0`은 잘못된 값 확인

2. **수정 내용**:
   - `window.clearCount = response.clear_count`로 전역 변수에 저장
   - `const clearCount = window.clearCount || 0`로 올바른 값 사용
   - 불필요한 디버깅 로그 정리

3. **수정된 파일**:
   - `/var/www/html/crossword-projects/src/main/resources/static/game.html`
     - `loadTemplate` 함수에서 `window.clearCount` 전역 변수 설정
     - `updateGameInfo` 함수에서 `window.clearCount` 사용

#### 결과
- ✅ 레벨 게이지가 레벨 클리어 횟수를 정확하게 표시
- ✅ 게이지 형식: ■□□ (1/3) - 형광 녹색으로 채워진 세그먼트
- ✅ 정답 개수와 레벨 클리어 횟수 구분 완료

#### 테스트 결과
- 게스트 ID `guest_1758994129146`로 테스트
- 레벨 2에서 1번 클리어한 상태에서 게이지 1/3 정상 표시 확인
- 콘솔 로그에서 `clearCount: 1` 정상 출력 확인
