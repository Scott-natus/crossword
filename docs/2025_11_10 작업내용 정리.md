# 2025-11-10 작업내용 정리

## Phase 1: 데이터베이스 관리 기능 (2025-10-31)

### 작업 개요
- 관리자 페이지에 개발서버와 운영서버 간 데이터베이스 동기화 기능 구현
- 테이블 목록에 테이블 코멘트(설명) 표시 기능 추가
- 테이블 동기화 시 코멘트도 함께 동기화하는 기능 구현
- 32개 테이블에 코멘트 추가

### 주요 구현 사항

#### 1. 데이터베이스 동기화 페이지 구현

**백엔드 구현:**
- `DatabaseSyncService.java`: 데이터베이스 동기화 서비스 클래스 생성
  - `getLocalTables()`: 로컬(개발) 서버 테이블 목록 조회
  - `getRemoteTables()`: 원격(운영) 서버 테이블 목록 조회
  - `syncTable()`: 테이블 동기화 메인 메서드
  - `getTableInfo()`: 테이블 정보 조회 (로우 개수, 최종 갱신일, 코멘트)
  - `createTableIfNotExists()`: 원격 서버에 테이블이 없으면 로컬 스키마로 생성
  - `getCreateTableStatement()`: PostgreSQL 스키마 기반 CREATE TABLE 문 동적 생성
  - `syncTableComment()`: 테이블 코멘트 동기화 메서드
  - `getTableData()`: 테이블 데이터 조회
  - `insertDataToRemote()`: 원격 서버에 데이터 일괄 삽입

**API 엔드포인트:**
- `GET /admin/api/db-sync/local-tables`: 개발 서버 테이블 목록 조회
- `GET /admin/api/db-sync/remote-tables`: 운영 서버 테이블 목록 조회
- `POST /admin/api/db-sync/sync/{tableName}`: 테이블 동기화 실행

**프론트엔드 구현:**
- `static/admin/db-sync/index.html`: 데이터베이스 동기화 관리 페이지
  - 좌측: 개발 서버 테이블 목록
  - 우측: 운영 서버 테이블 목록
  - 테이블 선택 후 동기화 버튼으로 실행
  - 테이블명, 로우 개수, 최종 갱신일, 테이블 코멘트 표시

#### 2. 테이블 코멘트 표시 기능 추가

**구현 내용:**
- 개발 서버 테이블 목록에 테이블 코멘트 표시
- 표시 형식: `테이블명 (코멘트)` 또는 코멘트가 없으면 `테이블명`만 표시
- `getTableInfo()` 메서드에서 `obj_description()` 함수로 코멘트 조회

**데이터베이스 코멘트 추가:**
- 32개 테이블에 코멘트 추가 완료
- 주요 테이블 코멘트:
  - `pz_words`: 퍼즐 단어 테이블 - 십자낱말 퀴즈에 사용되는 단어 저장
  - `pz_hints`: 퍼즐 힌트 테이블 - 퍼즐 단어에 대한 힌트 저장
  - `puzzle_levels`: 퍼즐 레벨 테이블 - 레벨별 난이도, 단어 수, 교차점 수, 시간 제한 등 설정
  - `boards`: 게시판 테이블 - 게시글 저장
  - `users`: 사용자 테이블 - 시스템 사용자 정보 저장
  - 기타 27개 테이블 코멘트 추가

#### 3. 인코딩 문제 해결

**문제:**
- 테이블 코멘트가 깨져서 표시됨 (예: `⑥ 蹂`)
- PostgreSQL JDBC 연결 시 UTF-8 인코딩 미지정

**해결 방법:**
- JDBC URL에 인코딩 파라미터 추가: `characterEncoding=utf8&useUnicode=true&stringtype=unspecified`
- 연결 생성 시 `SET client_encoding TO 'UTF8'` 명령 실행
- `application.properties`에 인코딩 설정 추가

#### 4. 테이블 동기화 시 코멘트 동기화 기능

**구현 내용:**
- `syncTableComment()` 메서드 추가
- 동기화 프로세스:
  1. 로컬 DB에서 테이블 코멘트 조회
  2. 원격 DB에 동일한 코멘트 적용
  3. 코멘트가 없으면 원격에서 제거
- SQL 인젝션 방지: 작은따옴표 이스케이프 처리 (`'` → `''`)
- 코멘트 동기화 실패해도 데이터 동기화는 계속 진행

#### 5. 동적 테이블 생성 기능

**구현 내용:**
- 원격 서버에 테이블이 없으면 자동 생성
- `getCreateTableStatement()`: PostgreSQL 스키마 기반 CREATE TABLE 문 동적 생성
  - 컬럼 정보 (이름, 데이터 타입, 길이, NULL 제약, 기본값)
  - PRIMARY KEY 제약조건
  - 시퀀스 생성 및 연결 (SERIAL 타입)
- PRIMARY KEY 조회 시 `key_column_usage` 사용 (이전 `constraint_column_usage` 오류 수정)

### 해결된 문제들

#### PRIMARY KEY 조회 오류
**문제**: `constraint_column_usage`에는 `ordinal_position` 컬럼이 없음  
**해결**: `key_column_usage` 사용으로 변경

#### 인코딩 문제
**문제**: 테이블 코멘트가 깨져서 표시됨  
**해결**: JDBC URL 인코딩 파라미터 추가 및 연결 시 UTF-8 설정

#### COMMENT ON TABLE 플레이스홀더 미지원
**문제**: PostgreSQL `COMMENT ON TABLE`은 PreparedStatement 플레이스홀더 미지원  
**해결**: Statement 사용 및 SQL 인젝션 방지를 위한 이스케이프 처리

#### 상세 작업 내역

**1. systemd 서비스 삭제**
- **문제**: systemd 서비스가 8080 포트를 점유하여 새로운 빌드 테스트 불가

  - systemd 재로드 (`systemctl daemon-reload`)
  - 8080 포트 해제 확인

**2. UNIQUE 제약조건 문제 해결**
- **문제**: `theme_daily_puzzles` 테이블의 `(theme, puzzle_date)` UNIQUE 제약조건으로 인한 INSERT 실패
  ```
  오류: 중복된 키 값이 "theme_daily_puzzles_theme_puzzle_date_key" 고유 제약 조건을 위반함
  Detail: (theme, puzzle_date)=(K-DRAMA, 2025-11-02) 키가 이미 있습니다.
  ```
- **원인**: `deactivateByThemeAndDate()`로 기존 레코드를 비활성화한 후 새로운 레코드를 INSERT하려 했지만, UNIQUE 제약조건 때문에 실패
- **해결**:
  - `ThemeDailyPuzzleRepository`에 `findByThemeAndPuzzleDate()` 메서드 추가 (isActive 무관)
  - `DailyPuzzleService.generateTodayPuzzle()` 로직 변경:
    - 기존 레코드 조회 (isActive 무관)
    - 존재하면 기존 레코드 업데이트
    - 없으면 새 레코드 생성
  ```java
  // 기존 퍼즐 조회 (UNIQUE 제약조건 때문에 theme + puzzle_date 조합은 하나만 존재)
  Optional<ThemeDailyPuzzle> existingPuzzleOpt = themeDailyPuzzleRepository
      .findByThemeAndPuzzleDate(theme, date);
  
  ThemeDailyPuzzle dailyPuzzle;
  if (existingPuzzleOpt.isPresent()) {
      // 기존 레코드 업데이트
      dailyPuzzle = existingPuzzleOpt.get();
      dailyPuzzle.setPuzzleId((Integer) puzzleData.get("puzzleId"));
      dailyPuzzle.setIsActive(true);
  } else {
      // 새 레코드 생성
      dailyPuzzle = new ThemeDailyPuzzle();
      // ... 필드 설정
  }
  ```

**3. Map 불변성 문제 해결**
- **문제**: `ThemePuzzleApiController.getTodayPuzzle()`에서 `Map.of()`로 생성한 불변 Map에 `put()` 시도로 인한 `UnsupportedOperationException`
  ```
  java.lang.UnsupportedOperationException: null
  at java.util.ImmutableCollections$AbstractImmutableMap.put
  ```
- **원인**: `Map.of()`는 불변(immutable) Map을 반환하므로 수정 불가
- **해결**: `HashMap`으로 변환하여 수정 가능하게 처리
  ```java
  // 응답에 테마와 날짜 정보 추가 (HashMap으로 변환하여 수정 가능하게)
  Map<String, Object> response = new HashMap<>(puzzleData);
  response.put("theme", theme);
  response.put("puzzleDate", puzzleDate.toString());
  return ResponseEntity.ok(response);
  ```

```java
for (테이블 in 테이블목록) {
    syncTableWithConnection(테이블, localConn, remoteConn);
}

// 모든 작업 완료 후 연결 닫기
```

  ```java
  public ResponseEntity<Map<String, Object>> saveGeneratedWord(
          @RequestBody Map<String, Object> request) {
      
      String word = (String) request.get("word");
      
      // 중복 체크
      Optional<PzWord> existing = pzWordRepository.findByWord(word);
      if (existing.isPresent()) {
          return ResponseEntity.badRequest()
              .body(Map.of("success", false, "message", "이미 존재하는 단어입니다."));
      }
      
      // 단어 저장
      PzWord newWord = new PzWord();
      newWord.setWord(word);
      newWord.setCategory((String) request.get("category"));
      newWord.setDifficulty((Integer) request.get("difficulty"));
      newWord = pzWordRepository.save(newWord);
      
      // 힌트 저장
      List<Map<String, Object>> hints = (List<Map<String, Object>>) request.get("hints");
      for (Map<String, Object> hintData : hints) {
          PzHint hint = new PzHint();
          hint.setWord(newWord);
          hint.setHintText((String) hintData.get("hint_text"));
          hint.setDifficulty((Integer) hintData.get("difficulty"));
          hint.setIsPrimary((Boolean) hintData.getOrDefault("is_primary", false));
          pzHintRepository.save(hint);
      }
      
      return ResponseEntity.ok(Map.of("success", true, "wordId", newWord.getId()));
  }
  ```

  ```java
  public void updateWord(Long puzzleId, String oldWord, Object newWord) {
      // 단어 ID 또는 단어 텍스트로 단어 조회
      PzWord word;
      if (newWord instanceof Integer) {
          word = pzWordRepository.findById((Integer) newWord)
              .orElseThrow(() -> new RuntimeException("단어를 찾을 수 없습니다."));
      } else {
          word = pzWordRepository.findByWord((String) newWord)
              .orElseThrow(() -> new RuntimeException("단어를 찾을 수 없습니다."));
      }
      
      // 기본 힌트 조회
      Optional<PzHint> primaryHint = pzHintRepository
          .findByWordAndIsPrimaryTrue(word);
      
      // puzzleData 업데이트
      Map<String, Object> puzzleData = getPuzzleData(puzzleId);
      List<Map<String, Object>> words = (List<Map<String, Object>>) puzzleData.get("words");
      List<Map<String, Object>> hints = (List<Map<String, Object>>) puzzleData.get("hints");
      
      // words 리스트에서 oldWord 찾아서 교체
      for (Map<String, Object> wordData : words) {
          if (oldWord.equals(wordData.get("word"))) {
              wordData.put("word", word.getWord());
              wordData.put("pz_word_id", word.getId());
              break;
          }
      }
      
      // hints 리스트에서 oldWord의 힌트 찾아서 교체
      if (primaryHint.isPresent()) {
          for (Map<String, Object> hintData : hints) {
              if (oldWord.equals(hintData.get("word"))) {
                  hintData.put("hint_text", primaryHint.get().getHintText());
                  hintData.put("pz_hint_id", primaryHint.get().getId());
                  break;
              }
          }
      }
      
      // puzzleData 저장
      savePuzzleData(puzzleId, puzzleData);
  }
  ```

  ```java
  public Map<String, Object> addStandardDictionaryHint(Integer wordId) {
      PzWord word = pzWordRepository.findById(wordId)
          .orElseThrow(() -> new RuntimeException("단어를 찾을 수 없습니다."));
      
      // 첫 번째 뜻풀이 조회
      String definition = standardDictionaryService.getFirstDefinition(word.getWord());
      
      if (definition == null || definition.isBlank()) {
          return Map.of("success", false, "message", "표준국어대사전에서 뜻풀이를 찾을 수 없습니다.");
      }
      
      // 중복 체크
      boolean exists = pzHintRepository.existsByWordAndHintText(word, definition.trim());
      if (exists) {
          return Map.of("success", false, "message", "이미 동일한 힌트가 존재합니다.");
      }
      
      // 힌트 저장
      PzHint hint = new PzHint();
      hint.setWord(word);
      hint.setHintText(definition.trim());
      hint.setDifficulty(3); // 어려움
      hint.setIsPrimary(false);
      hint.setHintType("STDICT");
      pzHintRepository.save(hint);
      
      return Map.of("success", true, "message", "표준국어대사전 힌트가 추가되었습니다.", "hint", hint);
  }
  ```

```javascript
function removeModalBackdrop() {
    // 남아있는 modal-backdrop 제거
    const backdrops = document.querySelectorAll('.modal-backdrop');
    backdrops.forEach(backdrop => backdrop.remove());
    
    // body에서 modal-open 클래스 제거
    document.body.classList.remove('modal-open');
    
    // body의 padding-right 스타일 제거 (Bootstrap이 추가하는 스크롤바 보정)
    document.body.style.paddingRight = '';
    
    // body의 overflow 스타일 복원
    document.body.style.overflow = '';
}
```

## Phase 8: 단어정제 모달 힌트 표시 문제 해결 (2025-11-10 추가)

### 작업 개요
- 단어정제 모달에서 힌트를 저장한 후 다시 모달을 열면 저장된 힌트가 보이지 않는 문제 해결
- API 응답 구조에 맞게 힌트 데이터를 읽어오도록 수정

### 문제 상황
- 단어정제 모달에서 힌트 추가/생성/대사전 힌트 추가 후 "확정하기" 버튼으로 저장
- 힌트는 정상적으로 저장됨 (데이터베이스 확인)
- 저장 후 다시 모달을 열면 저장된 힌트가 표시되지 않음

### 원인 분석
- 백엔드 API 응답 구조:
  - `data.data`: 단어 정보
  - `data.hints`: 힌트 목록 (최상위 레벨)
- 프론트엔드 코드에서 힌트 데이터 접근 방식 확인 필요

### 해결 방법

#### 1. 프론트엔드 힌트 표시 로직 개선

**파일**: `java-projects/src/main/resources/static/admin/words/index.html`

**변경 내용**:
- `openRefinementModal()` 함수에서 힌트 데이터를 안전하게 읽어오도록 수정
- `data.hints`가 배열인지 확인하고, 배열이 아니면 `data.data.hints`도 확인
- 빈 배열 기본값 설정

**구현**:
```javascript
// 힌트 정보 표시
console.log('API 응답 데이터:', data);
console.log('힌트 데이터:', data.hints);
// 힌트가 배열인지 확인하고 표시 (백엔드에서 최상위 레벨의 hints 키로 반환)
const hints = Array.isArray(data.hints) ? data.hints : (data.data && Array.isArray(data.data.hints) ? data.data.hints : []);
console.log('표시할 힌트:', hints);
displayHintsForRefinement(hints);
```

#### 2. 백엔드 API 응답 구조 확인

**파일**: `java-projects/src/main/java/com/example/board/service/WordManagementService.java`

**확인 내용**:
- `getWordDetail()` 메서드에서 힌트 목록을 `result.put("hints", hintsList)`로 반환
- 힌트는 최상위 레벨의 `hints` 키로 반환됨
- 단어 정보는 `result.put("data", wordData)`로 반환

**응답 구조**:
```json
{
  "success": true,
  "data": {
    "id": 123,
    "word": "가건물",
    ...
  },
  "hints": [
    {
      "id": 456,
      "hint_text": "힌트 내용",
      "difficulty": 2,
      "is_primary": true
    }
  ]
}
```
