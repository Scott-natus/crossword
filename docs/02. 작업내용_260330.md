# 2026. 03. 30. 작업 내역

## 1. 개요
Gemini API 연동 중 발생한 `400 Bad Request` 에러 해결 및 포트 꼬임으로 인한 `503 Service Unavailable` 복구 작업 수행.

## 2. 작업 상세 내용

### **가. Gemini API 연동 및 모델 업데이트**
- **문제**: 이전 사용하던 API Key 만료 및 구형 모델(`gemini-2.5-flash-lite`) 사용으로 인한 요청 실패.
- **수행 내용**: 
    - 신규 API Key 발급 및 `application.properties` 반영.
    - `GeminiService.java` 내의 API 호출 URL을 최신 모델(`gemini-flash-latest`)로 강제 변경.
    - Spring Boot의 `@Value` 어노테이션에 의한 설정 덮어쓰기 문제를 해결하기 위해 필드 직접 수정.

### **나. 503 서비스 장애 복구 (8081 & 8080)**
- **문제**: 포트 리스닝 프로세스가 꼬여 `503 Service Unavailable`이 발생.
- **수행 내용**:
    - **프로세스 킬**: `pkill -9 -f crossword` 및 `gradle` 명령으로 충돌 중인 프로세스 종료.
    - **프로젝트 빌드 초기화**: `./gradlew clean`으로 이전 빌드 결과물을 모두 삭제.
    - **개별 프로젝트 재빌드 및 기동**:
        - 8081 서비스: `clean assemble` 후 단독 기동 (포트 8081).
        - 8080 서비스: `clean assemble` 후 단독 기동 (포트 8080).
- **결과**: 두 서비스 모두 **200 OK** 응답 확인 및 원복 완료.

### **다. 프로젝트 구조 정리**
- **문서화**: `/var/www/html/docs` 폴더를 생성하여 시스템 아키텍처 및 오늘자 작업 내역을 `.md` 파일로 정리.

### **라. Gemini API 설정 최적화 및 보안 강화**
- **문제**: API 키와 URL이 환경 변수(`~/.bashrc`)에 고정되어 관리가 어렵고, `application.properties`의 설정보다 우선순위가 높아 유연한 변경이 불가능했던 점을 개선.
- **수행 내용**:
    - **`.env` 아키텍처 도입**: 각 프로젝트 루트(`/var/www/html/crossword-projects`, `/var/www/html/java-projects`)에 `.env` 파일을 생성하여 API Key와 URL을 격리 관리.
    - **환경 변수 정리**: `~/.bashrc`에 등록된 기존 `GEMINI_API_KEY`, `GEMINI_API_URL` 설정을 제거하여 프로젝트별 독립성 확보.
    - **Spring Boot 설정 연동**: `application.properties`에 `spring.config.import` 옵션을 사용하여 `.env` 파일을 자동으로 읽어오도록 구성하고, 중복된 하드코딩 값을 `${GEMINI_API_KEY}` 등의 변수로 대체.
    - **코드 리팩토링**: 
        - `GeminiService.java`: 하드코딩된 URL을 제거하고 `@Value`를 통해 설정을 주입받도록 수정.
        - `HintGeneratorManagementService.java`: 기본 모델 URL을 최신(`gemini-flash-latest`)으로 보정.
- **결과**: 서비스 재시작 후 스케줄러를 통한 힌트 생성 성공 및 DB 정상 반영 확인.
