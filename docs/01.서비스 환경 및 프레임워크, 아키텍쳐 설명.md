# 서비스 환경 및 프레임워크, 아키텍처 설명

## 1. 개요
본 시스템은 대규모 서비스를 위한 다중 모듈 기반의 Java 웹 애플리케이션으로, 게시판 시스템(Board System)과 크로스워드 프로젝트(Crossword Project)가 독립적인 프로세스로 운영되는 구조입니다.

## 2. 기술 스택
- **언어**: Java 17+
- **프레임워크**: Spring Boot 3.x
- **빌드 도구**: Gradle (다중 모듈 및 독립 프로젝트 구성)
- **데이터베이스**: MySQL / MariaDB (JPA 기반 데이터 관리)
- **캐싱 및 서버**: Redis (Session 관리용), Apache/Nginx (Reverse Proxy)

## 3. 서비스 구성 (아키텍처)
시스템은 포트 기반으로 서비스가 분리되어 있습니다.

| 서비스명 | 포트 | 주요 기능 |
| :--- | :--- | :--- |
| **Board System (8080)** | 8080 | 게시판 관리, 사용자 커뮤니티 기능 제공 |
| **Crossword Project (8081)** | 8081 | 크로스워드 게임 로직, Gemini AI 기반 힌트 자동 생성 |

## 4. 핵심 모듈 설명
- **Gemini AI Integration**: `GeminiService`를 통해 Google Gemini API와 연동되어 크로스워드 단어에 대한 힌트를 자동으로 생성 및 저장합니다.
- **Scheduler**: `HintGenerationScheduler`가 주기적으로 힌트가 없는 단어를 확인하여 AI 서비스를 호출합니다.

## 5. 배포 환경
- **OS**: Linux (Ubuntu 기반)
- **웹 서버 서버 경로**: `/var/www/html`
- **프로젝트 소스 경로**:
    - `/var/www/html/java-projects` (8080 서비스)
    - `/var/www/html/crossword-projects` (8081 서비스)

## 6. 로그 설정

애플리케이션·API·프레임워크 로그는 **Logback** 으로 통합하며, **모든 파일 로그의 기본 디렉터리**는 아래와 같다.

| 항목 | 값 |
| :--- | :--- |
| **로그 루트 디렉터리** | `/home/ubuntu/crossword/log` |
| **디렉터리 생성** | 저장소에 `log/.gitkeep` 포함, 기동 스크립트에서 `mkdir -p` 수행 |
| **경로 변경 (선택)** | JVM 옵션 `-DLOG_DIR=/원하는/경로` — `logback-spring.xml` 의 `${LOG_DIR}` 에 반영 |

### 6.1 Logback 구성 파일

| 서비스 | 설정 파일 |
| :--- | :--- |
| 게시판 (8080) | `java-projects/src/main/resources/logback-spring.xml` |
| 크로스워드 (8081) | `crossword-projects/src/main/resources/logback-spring.xml` |

공통 동작:

- **콘솔(CONSOLE)** + **파일(FILE)** 이중 출력.
- **파일**은 `RollingFileAppender` + `TimeBasedRollingPolicy` 로 **날짜 단위 롤링**.
- **보관 일수**: `maxHistory` **5** — 5일이 지난 아카이브 파일은 Logback 이 자동 삭제.

### 6.2 애플리케이션 로그 파일명

| 서비스 | 당일 활성 파일 | 날짜별 아카이브 패턴 |
| :--- | :--- | :--- |
| 게시판 (8080) | `board-application.log` | `board-application.yyyy-MM-dd.log` |
| 크로스워드 (8081) | `crossword-application.log` | `crossword-application.yyyy-MM-dd.log` |

로그 레벨·패키지별 상세 레벨은 각 모듈의 `application.properties` 의 `logging.level.*` 로 조정한다. 파일 경로·롤링은 `logback-spring.xml` 이 담당하므로, **`logging.file.name` 등 Spring Boot 파일 로그 속성은 사용하지 않는다.**

### 6.3 Gradle `bootRun` 콘솔 캡처

`restart-crossword.sh` 로 기동할 때 Gradle 이 출력하는 표준출력·표준에러는 애플리케이션 로그와 별도로 아래 파일에 **추가(append)** 된다.

| 구분 | 기본 파일 (환경변수로 덮어쓰기 가능) |
| :--- | :--- |
| 게시판 `bootRun` | `$CROSSWORD_LOG_DIR/gradle-board-boot.log` (`BOARD_BOOT_LOG`) |
| 크로스워드 `bootRun` | `$CROSSWORD_LOG_DIR/gradle-crossword-boot.log` (`CROSSWORD_BOOT_LOG`) |

- **`CROSSWORD_LOG_DIR`**: 미지정 시 스크립트 위치 기준 `.../crossword/log` (`mkdir -p` 됨).

스크립트 종료 시 **`gradle-*.log` 중 수정 시각이 5일이 지난 파일**만 `find … -delete` 로 정리한다. Logback 이 생성한 `board-application.*` / `crossword-application.*` 일별 파일은 **`maxHistory=5`** 로 별도 관리되므로, 위 `find` 는 Gradle 로그 위주 정리용이다.

### 6.4 확인 명령 예시

```bash
# 실시간 애플리케이션 로그
tail -f /home/ubuntu/crossword/log/board-application.log
tail -f /home/ubuntu/crossword/log/crossword-application.log

# Gradle 기동 로그
tail -f /home/ubuntu/crossword/log/gradle-board-boot.log
tail -f /home/ubuntu/crossword/log/gradle-crossword-boot.log
```

## 7. 시크릿·환경 변수 (`.env`)

비밀번호·API 키는 **저장소 루트**의 **한 개 `.env`** 에만 둔다 (`/home/ubuntu/crossword/.env`). `spring.config.import` 는 다음 순서다.

```
optional:file:../.env[.properties],optional:file:.env[.properties]
```

즉 **루트 `.env`를 먼저** 읽고, 필요하면 **각 모듈 폴더의 `.env`** 로 덮어쓸 수 있다. 루트 `.env` 에는 예를 들어 아래가 들어간다.

| 키 | 용도 |
| :--- | :--- |
| `GEMINI_API_KEY`, `GEMINI_API_URL` | 게시판·크로스워드 공통 (Gemini 연동) |
| `NATUS_DB_PASSWORD` | 로컬 PostgreSQL `natus_user` |
| `REMOTE_DB_PASSWORD` | 게시판 원격 DB 동기화(선택) |

루트에 **`.env.example`** 이 있으면 복사해 `.env` 를 만든다: `cp .env.example .env` (`.env` 는 `.gitignore` 대상).

원격 DB 호스트 등 비밀 아닌 값은 `application.properties` 의 `remote.db.*` 기본값·`${REMOTE_DB_*}` 로 둔다.
