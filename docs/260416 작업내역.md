# 2026. 04. 16. 작업 내역

## 1. 개요

하네스 **Engine-0**(트렌드 수집)·**Engine-1**(트렌드 → K-CrossWord 일일 테마 5종 매핑 v1) 구축을 진행했다. YouTube 수집은 **검색·필터** 기반으로 전환하고, Instagram **릴스 수집**을 추가했다. DB 적재·Gemini 메타데이터 보강·Crontab 일일 배치까지 단계별로 정리했다.

**구현 코드 위치**: 저장소 루트의 `harness-factory/` (본 `crossword` Git에 포함됨.)

## 2. 작업 상세 내용

### 가. Engine-0 스캐너 (YouTube·Instagram)

- **YouTube**: 검색 URL, `hl=en`, 쿠키 선택, UI 필터 옵션, 결과를 `engine0_shorts.json` 등으로 저장.
- **Instagram 릴스**: `reels_scanner.py`로 `engine0_reels.json` 수집.
- **Phase 1 — PostgreSQL (`engine0_trends`)**: `ingest_to_postgres.py`, `db_env.py`로 JSON 적재. `crossword/.env`의 `NATUS_DB_*`를 `PG*` 환경 변수로 매핑.
- **Phase 2 — Gemini 메타데이터**: `enrich_gemini.py` (`google-genai`, 기본 `gemini-flash-latest`). `--dry-run` 시 DB 미연결·샘플만 Gemini 호출.
- **Phase 3 — 운영**: `daily_engine0_run.sh`, `report_engine0_summary.py`, `requirements-db.txt` / `requirements-ai.txt`, `crontab.example` (**매 3시간** `0 */3 * * *`).

### 나. DB 환경·연결 이슈 해결

- **`gemini-2.0-flash` 404**: `google-genai` + **`gemini-flash-latest`** 로 전환.
- **`psql` / Python DB**: `role "ubuntu" does not exist`, `DATABASE_URL` 없음 → **`with_db.sh`** 또는 `NATUS_*`로 연결.
- **`password authentication failed for user "natus_user"` (Python)**: 셸에 남은 잘못된 `PGPASSWORD` 방지를 위해 **`db_env`에서 `PGPASSWORD`를 `NATUS_DB_PASSWORD`로 항상 설정**하도록 수정.

### 다. Engine-1 (트렌드 → 테마 매핑 v1)

- **스키마**: `sql/001_engine1_trend_theme_map.sql` — `engine1_trend_theme_map` (FK `engine0_trends`).
- **스크립트**: `map_trend_themes.py` — 키워드 규칙으로 테마 매핑, `--dry-run` / `--force` / `--limit`.
- **실행 보조**: `with_db.sh` — `.env`에서 `DATABASE_URL` 구성 후 `psql` (수동 비밀번호 불필요).
- **결과**: 매핑 테이블 **COUNT = 57** 저장 확인. 다수 행은 제목만 있거나 키워드 미매칭으로 `Korean` 등으로 분류, 일부 `K-DRAMA`(예: `ost` 키워드) 매칭.
