### Java 클래스 명명 규칙

- **Service**: `[모듈명]Service.java` (예: `MemberService.java`)
- **Repository**: `[모듈명]Repository.java` (예: `MemberRepository.java`)
- **Entity**: `[모듈명].java` (예: `Member.java`)

### 패키지 구조

## 🔄 디렉토리 간 연관 관계

### 데이터 흐름
```
members → product → orders → sales
    ↓         ↓        ↓        ↓
    └─────────┴────────┴────────┘
              ↓
         analyzer
              ↑
    promotion, organize
```

### 주요 연관 관계
- **members ↔ product**: 회원별 제품 구매 이력
- **product ↔ orders**: 제품별 발주 정보
- **orders ↔ sales**: 발주별 매출 정보
- **promotion ↔ sales**: 프로모션별 매출 효과
- **organize ↔ sales**: 조직별 매출 현황
- **analyzer**: 모든 모듈의 데이터를 종합 분석
