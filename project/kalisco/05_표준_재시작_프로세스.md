## 📋 수동 재시작 (단계별)

### 1단계: 서비스 다운
```bash
# systemd 서비스 중지
sudo systemctl stop kalisco-service

# JAR 프로세스 종료 (혹시 실행 중인 경우)
pkill -f "java.*kalisco.*jar"

# 포트 해제 확인
netstat -tlnp | grep 9091
```

### 2단계: 빌드 (JAR 파일 생성)
```bash
# 프로젝트 디렉토리로 이동
cd /var/www/html/project/kalisco

# Gradle 빌드 실행
./gradlew clean bootJar

# 빌드 성공 확인
ls -lh build/libs/kalisco-0.0.1-SNAPSHOT.jar
```

**빌드 실패 시:**
- 컴파일 오류 확인: `./gradlew compileJava`
- 의존성 문제 확인: `./gradlew dependencies`
- 로그 확인: 빌드 출력 메시지 확인

### 3단계: JAR 파일로 서비스 시작
```bash
# systemd 서비스 시작
sudo systemctl start kalisco-service

# 서비스 상태 확인
sudo systemctl status kalisco-service

# 포트 리스닝 확인
netstat -tlnp | grep 9091
```

## ⚠️ 주의사항

### 1. 빌드 실패 시
- 서비스를 시작하지 않음
- 빌드 오류를 먼저 해결해야 함
- 컴파일 오류, 의존성 문제 확인

### 2. 포트 충돌
- 9091 포트가 이미 사용 중이면 서비스 시작 실패
- 기존 프로세스를 완전히 종료해야 함
- `pkill -f "java.*kalisco.*jar"`로 강제 종료

### 3. JAR 파일 누락
- 빌드가 실패하면 JAR 파일이 생성되지 않음
- 서비스 시작 전 JAR 파일 존재 확인 필요
- `ls -lh build/libs/kalisco-0.0.1-SNAPSHOT.jar`

### 4. 메모리 부족
- JAR 파일 실행 시 메모리 사용량 확인
- 필요시 JVM 옵션 추가: `-Xmx512m -Xms256m`

## 🛠️ 유용한 명령어

### 서비스 관리
```bash
# 서비스 시작
sudo systemctl start kalisco-service

# 서비스 중지
sudo systemctl stop kalisco-service

# 서비스 재시작 (빌드 없이)
sudo systemctl restart kalisco-service

# 서비스 상태 확인
sudo systemctl status kalisco-service

# 서비스 자동 시작 활성화
sudo systemctl enable kalisco-service

# 서비스 자동 시작 비활성화
sudo systemctl disable kalisco-service
```

```bash
./gradlew clean bootJar

# 테스트 제외 빌드
./gradlew bootJar -x test

# 빌드만 실행 (서비스 시작 없이)
./gradlew bootJar
```

```bash
# JAR 프로세스 종료
pkill -f "java.*kalisco.*jar"
```
